标题：全局错误码（微信客服 API）

================================================================================
全局错误码
================================================================================

说明：企业应用每次调用接口时，可能获得正确或错误的返回码，企业可以根据返回码信息调试接口、排查错误。

注意：
  - 开发者的程序应该根据 errcode 判断出错情况，而不应依赖 errmsg 做字符串匹配（errmsg 可能会调整）。
  - 若请求参数不符合 JSON 规范，可能导致企业微信解析参数不完整，此时 errmsg 可能出现 “Warning: wrong json format.”，需重新检查 JSON 合法性。

| 错误码 | 错误说明 | 排查方法 |
| --- | --- | --- |
| -1 | 系统繁忙 | 服务器暂不可用，建议稍候重试。建议重试次数不超过 3 次。 |

| 40001 | 不合法的 secret 参数 | secret 在开发配置/配置服务页面可查看 |
| 40004 | 不合法的媒体文件类型 | 不满足系统文件要求；参考上传的媒体文件限制 |
| 40005 | 不合法的 type 参数 | 合法 type 取值参考上传临时素材 |
| 40006 | 不合法的文件大小 | 系统文件要求，参考上传的媒体文件限制 |
| 40007 | 不合法的 media_id 参数 | 查看帮助 |
| 40008 | 不合法的 msgtype 参数 | 合法 msgtype 取值参考消息类型 |
| 40009 | 上传图片大小不是有效值 | 图片大小系统限制，参考上传的媒体文件限制 |
| 40011 | 上传视频大小不是有效值 | 视频大小系统限制，参考上传的媒体文件限制 |
| 40014 | 不合法的 access_token | 查看帮助 |
| 90207 | 不合法的小程序 appid | - |
| 95000 | 不合法的 open_kfid | - |
| 95001 | 发送客服消息次数限制 | 用户主动发消息后企业最多发 5 条；用户继续发消息后可再次下发 |
| 95002 | 发送客服消息时间限制 | 用户主动发消息后的 48 小时内可调用发送接口 |
| 95003 | 发送客服消息可接待客户咨询数限制 | 企业未验证或未绑定视频号 |
| 95004 | open_kfid 不存在 | 跨企业使用，或对已删除账号操作 |
| 95005 | 客服账号数超过限制 | 不能超过最大数量限制，且至少保留一个 |
| 95006 | 不合法的客服账号名 | - |
| 95007 | 不合法的 msgtoken | msgtoken 有时效性，可能失效或非法 |
| 95008 | 菜单消息菜单项个数超过上限 | 目前最多 10 个菜单项 |
| 95009 | 不合法的菜单消息菜单项类型 | - |
| 95011 | 已在企业微信使用微信客服 | - |
| 95017 | 企业内部调用 API 时 API 开关处于关闭状态 | - |
| 95022 | location_type 非法 | - |
| 95024 | 48 小时内无客户主动发消息记录，不能发送此消息 | 点击菜单消息的回复不算 |
| 95025 | 48 小时内最多只能发送 1 条名片消息 | - |
| 95026 | 客服账号使用率较低，暂无法创建 | 请改善后再试 |
| 95027 | 企业未认证，仅可创建 10 个客服账号 | 认证企业后可继续创建 |
| 95028 | 撤回消息的 msgid 不存在 | - |
| 95029 | 撤回消息的时效已过期 | 只能撤回 2 分钟内通过 API 发送的消息 |

排查方法（官方摘录）

错误码 40007（不合法媒体文件）：
  1）media_id 获取方式是否存在；上传临时素材生成的 media_id 有效期 3 天
  2）媒体类型需符合接口要求（例如发图片消息不能用音频 media_id）

错误码 40014（access_token 错误）：
  1）access_token 获取方式是否正确
  2）是否过期，可重新 gettoken
