来源：https://developer.work.weixin.qq.com/document/path/90968
标题：加解密方案说明（企业微信；微信客服回调算法一致）
归档说明：本文件为「回调消息体 AES 加解密与签名校验」的离线摘录，便于与 93304-开发指引.txt 对照实现。完整示例与多语言加解密库下载以官方页面为准。

================================================================================
概述
================================================================================

企业微信在推送消息给企业时，会对消息内容做 AES 加密，以 XML 格式 POST 到企业应用 URL。企业被动响应时也需要加密并以 XML 返回。

术语：
  EncodingAESKey：43 字符，从 a-z A-Z 0-9 选取；为 AESKey 的 Base64 编码。解码后为 32 字节 AESKey。
    AESKey = Base64_Decode(EncodingAESKey + "=")
  msg_signature：消息签名，用于验证请求是否来自企业微信（防伪造）
  AESKey：AES 密钥，32 字节；AES 采用 CBC 模式；数据 PKCS#7 填充至 32 字节倍数；IV 取 AESKey 前 16 字节
  msg：明文消息体（XML）
  msg_encrypt：明文 msg 加密后的 Base64

官方建议：优先使用官方提供的各语言加解密库（含 VerifyURL / DecryptMsg / EncryptMsg 三类能力）。

ReceiveId（加解密库初始化参数）
  在不同场景含义不同：
  1）企业应用回调：corpid
  2）第三方事件回调：suiteid
  3）个人主体第三方应用：空字符串

微信客服回调：按企业自建接入处理时，ReceiveId 使用企业 corpid（与 93304 说明一致）。

================================================================================
VerifyURL（验证回调 URL）
================================================================================

入参（来自 URL 与参数）：
  sMsgSignature, sTimeStamp, sNonce, sEchoStr（echostr 必须 urldecode 后再用）

输出：
  sReplyEchoStr：解密后的明文，用于 HTTP 响应体，必须原样返回（不要加引号等）

================================================================================
DecryptMsg（解密 POST 包）
================================================================================

入参：
  sMsgSignature, sTimeStamp, sNonce, sPostData（整个 POST 数据）

输出：
  sMsg：解密后的明文 XML

================================================================================
EncryptMsg（加密被动回复）
================================================================================

入参：
  sReplyMsg（明文 XML）, sTimeStamp, sNonce（由调用方生成）

输出：
  sEncryptMsg：加密后的 XML 包

================================================================================
消息体签名校验（原理）
================================================================================

计算签名：
  dev_msg_signature = sha1( sort(token, timestamp, nonce, msg_encrypt) 拼接成字符串后做 sha1 )
说明：
  sort：四个参数值按字母字典序排序后拼接
  sha1 结果编码为可见字符：每字节以 %02x 十六进制输出，全小写
比较 dev_msg_signature 与 msg_signature 是否相等。

被动回复时企业也需用同样方式生成签名传给企业微信。

================================================================================
明文加密 / 密文解密（原理摘要）
================================================================================

拼接明文字符串：
  rand_msg = random(16B) + msg_len(4B, 网络字节序) + msg + receiveid

加密：
  msg_encrypt = Base64_Encode( AES_Encrypt(rand_msg) )

解密：
  aes_msg = Base64_Decode(msg_encrypt)
  rand_msg = AES_Decrypt(aes_msg)
  去掉前 16 随机字节；读取 4 字节 msg_len；截取 msg_len 长度得到 msg；剩余为 receiveid；校验 receiveid/msg_len。
